# Meta日報 前日分記録スクリプト

Meta日報のテスト用シートで、Lステップ由来の実CVとMeta広告数値を日次で反映するための運用メモです。MacBook側で確認・修正するときも、このファイルと `scripts/google-apps-script/` 配下を見れば復旧できるようにしています。

## 現在の対象

テスト用シート:

https://docs.google.com/spreadsheets/d/1enOPyfKDTw5Xf5m6rsLznZRqLfTXV9ZkaWvlEkhY1R4/edit

確認用・テスト用シートは今後コピーでURLが変わる可能性があります。その場合は Apps Script 冒頭の `META_DAILY_SHEET_ID` または `META_DAILY_CONFIG.spreadsheetId` を新しいスプレッドシートIDへ差し替えてください。

## 実装ファイル

- `scripts/google-apps-script/meta-daily-yesterday-recorder-lite.gs`
  - Apps Scriptへ直接貼り付ける用の軽量版。
  - 今のテストでは基本こちらを使う。
- `scripts/google-apps-script/meta-daily-yesterday-recorder.gs`
  - メニュー、指定日実行、トリガー、RAW同期メニューを含む通常版。
- `scripts/google-apps-script/appsscript.json`
  - Apps Scriptマニフェスト。必要スコープの確認用。

## 毎日の運用

1. Lステップから前日分CSVを抽出する。
2. `昨日のデータ` タブの末尾に追記する。
3. Apps Scriptで `recordYesterdayMetrics` を実行する。
4. 各キャンペーンタブの前日行を確認する。

重要: `昨日のデータ` は過去分を消さず、末尾に追加していく運用です。過去日の行を消すと、過去日の実CVも消えます。元シートでは行数が多いため、このタブを非表示にしています。

## 日付の切り替え

日付の切り替えは、各キャンペーンタブのA列の日付で行います。

例:

- 2026-07-08が57行目なら、実CVは `$A57` を見る。
- 2026-07-09が58行目なら、実CVは `$A58` を見る。

実CV列には以下の形の `COUNTIFS` が入ります。行番号だけが日付行に合わせて変わります。

```gs
=IFERROR(COUNTIFS('昨日のデータ'!$C$2:$C,">="&INT($A57),'昨日のデータ'!$C$2:$C,"<"&INT($A57)+1,'昨日のデータ'!$E$2:$E,"*"&$I$2&"*",'昨日のデータ'!$B$2:$B,"<>"),0)
```

このため、`昨日のデータ` に複数日分が混在していても、各行の日付に一致するデータだけを拾います。

## 記録される列

各キャンペーンタブの前日行に、以下を書き込みます。

- `B`: インプレ
- `C`: クリック
- `D`: 費用
- `H`: CV
- `I`: 実CVの `COUNTIFS` 関数

以下の計算列は、該当行の式を整えます。

- `E`: CTR
- `F`: CPC
- `G`: CPM
- `J`: CVR
- `K`: CPA

## 実CVの照合ルール

実CVはタブ名ではなく、各タブの `I2` と `昨日のデータ` E列の登録経路を部分一致で照合します。

`I2` はGASが自動で末尾コードへ更新します。

例:

- `N117` タブ: `I2 = N117`
- `N118 ` タブ: `I2 = N118`
- `（セミナー誘導）ダイエット005` タブ: `I2 = ダイエット005`
- `ダイエット05` タブ: `I2 = ダイエット05`

`J2` は使わないため、GAS実行時に空欄へ戻します。

## RAW_ADSとMeta数値

`B:D` と `H` は `RAW_ADS` から取得します。

現在の軽量版は、`RAW_ADS` に対象日の広告データがある場合だけ `B:D` と `H` を書き込みます。対象日のRAWがない場合、0ではなく空欄に戻します。

今後の予定:

1. テスト用で数日、実CVが正しく残るか確認する。
2. 問題なければ `B:D` と `H` をMeta管理画面/API由来で反映できるように実装する。
3. 数日テスト継続する。
4. 全体に問題がなければ、本番環境へ Apps Script と必要な関数を移動する。

## トラブル時の確認

実CVが0のまま:

- `昨日のデータ` の列が `A:ID / B:氏名 / C:日付 / D:シナリオ / E:登録経路` になっているか確認する。
- `昨日のデータ` C列の日付が `2026-07-08 0:25:09` のように日付として認識できる形か確認する。
- 対象タブの `I2` が、`昨日のデータ` E列の登録経路に含まれているか確認する。
- 例: E列が `広告_FB_ベルラス_ダイエット05` なら、`I2 = ダイエット05` で拾える。

`B:D` と `H` が空欄:

- `RAW_ADS` に対象日付のデータがない可能性が高い。
- 現在は0を書かず空欄に戻す仕様。
- Meta連携実装後は、この欄が自動で埋まる想定。

過去日の実CVが消えた:

- `昨日のデータ` から過去日の行を削除した可能性がある。
- 関数が `昨日のデータ` を参照しているため、履歴は削除しない。

## MacBookで修正する時

1. GitHubから最新を取得する。
2. このMarkdownを読む。
3. Apps Scriptに貼る場合は、まず `meta-daily-yesterday-recorder-lite.gs` を使う。
4. テスト用シートURLが変わっている場合は、スプレッドシートIDを差し替える。
5. Apps Scriptで `recordYesterdayMetrics` を実行して確認する。

Mac側で本番移行する場合も、先にテスト用で数日確認してから本番へ移してください。
